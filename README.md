# 📊 채용 플랫폼 유저 행동 로그 분석 및 서비스 활성화 전략 수립
> **데이터 기반의 핵심 지표(Baseline) 정의를 통한 유저 행동 진단 및 전환 최적화**

## 1. 📝 프로젝트 배경 (Background)
* **현황**: 플랫폼 내 유저 활동이 존재하지만, 어떤 행동이 서비스 안착과 최종 지원으로 이어지는지에 대한 정량적 기준 부재
* **목표**: 유저 로그를 분석하여 서비스의 건강도를 측정할 **핵심 지표를 정의**하고, 이를 바탕으로 유저 이탈을 막기 위한 구체적인 활성화 방안 도출

## 2. 🎯 분석의 기준점: 핵심 지표 선정 (Metrics Definition)
> **전략 수립을 위해 분석의 기준이 될 '데이터 지표'를 먼저 정의하는 과정**
* **지표 탐색**: 유저 세션 로그를 바탕으로 '공고 조회', '이력서 수정', '북마크' 등 주요 이벤트 간의 상관관계 분석
* **기준 확립**: 최종 목표인 '지원단계 진입'과 가장 밀접하게 연결된 행동을 탐색하여, 유저가 서비스에 정착했음을 판단하는 **기초 지표(Baseline) 선정**
* **의의**: 파편화된 데이터를 하나의 핵심 지표로 수렴시켜, 마케팅 및 운영 팀이 집중해야 할 명확한 목표 지점 제시

## 3. 🔍 주요 분석 결과 (Key Insights)
* **퍼널 분석 (Funnel)**: 가입부터 지원까지의 단계별 전환율을 산출하여, 유저가 가장 많이 이탈하는 '병목 구간'을 정량적으로 특정
* **행동 패턴 분석**: 핵심 지표를 달성한 그룹과 그렇지 못한 그룹 간의 결정적 행동 차이(예: 검색 필터 사용 빈도, 북마크 활용 주차 등) 발견
* **전환 요인 탐색**: 최종 지원에 성공한 유저들이 공통적으로 거치는 '사전 행동 경로' 파악

## 4. 💡 비즈니스 활성화 제안 (Action Plan)
* **지표 개선 안**: 정의된 핵심 지표를 높이기 위한 서비스적 보완점 제안 (예: 이력서 작성 프로세스 개선, 북마크 유도 UI 배치)
* **개인화 전략**: 유저의 활동 이력에 따라 이탈 가능성을 예측하고, 적시에 푸시 알림이나 맞춤형 공고를 제안하는 시스템적 접근
* **운영 효율화**: 분석 결과에 근거하여 리소스(광고, 프로모션)를 집중해야 할 고효율 유저 세그먼트 제안

## 5. 🛠 기술 스택 및 나의 기여 (Tech Stack & Contribution)
* **언어/환경**: Python (Pandas, Numpy, Seaborn), Google Colab
* **핵심 역할**:
  * 팀원 간 산재된 분석 코드를 하나로 통합하고 데이터 정합성을 검증하는 **'코드 파이프라인 통합'** 수행
  * 단순 현황 파악을 넘어, 전략 수립의 근거가 되는 **'핵심 지표 선정 논리'** 구축
  * 분석 데이터 마트 생성 및 시각화를 통한 최종 전략 보고서 작성
